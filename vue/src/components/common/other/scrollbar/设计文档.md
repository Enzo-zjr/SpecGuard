## 示例
- 自定义滚动条的实现

## 组件功能点分析
- 常规功能点分析
    - 内容体
        - 滚轮滚动，页面内容滚动，相应的滚动条滚动
    - 滚动条
        - 拽着滚动条，可以上面滚动， 相应的内容体滚动
        - 点击滑道的某个位置，滚动条滚到目标位置，相应的内容体滚动到目标位置

- 扩展功能点分析
    - 滚动条可以设置默认是否隐藏
        - 默认隐藏，鼠标移入显示
    - 滚动条可以设置最小尺寸
        - 避免内容太多，滚动条太小的情况出现     
    - 可监听内部变化
        - 内容变化，自动更新滚动条
    - 可以主动更新滚动条的位置
       - update 方法
       - 需要考虑动画过程
    - 可以主动刷新滚动条
       - refresh
    - 对外暴露事件
        - scroll 滚动事件
        - scrolltoupper， 上啦，滚动到顶部/左边时触发  
        - scrolltolower， 下滚触底，滚动到底部/右边时触发  
- 核心原理分析  
    - 隐藏原生滚动条，创建模拟滚动条，根据当前滚动条的状态，设置模拟滚动条的状态

## 组件具体实现
### 1.组件结构设计
- 滚动条容器
    - 滚动条内容容器（出现滚动条的容器）
         - 内容装载容器，所有内容全部装载在此容器中
    - 竖向滚动条容器
        - 滚动条
    - 横向滚动条容器
        - 滚动条
### 2.组件对外接口设计
            
#### 2.1 事件接口
- 组件加载完成事件 
    -  ready    
- 常规事件
    - scroll     
    - 滚动条只有一个事件，那就是滚动（scroll）事件
- 扩展事件
    - scrollToUpper 
    - scrollToLower 
    -  scrolltoupper 和  scrolltolower 都是wheel事件，在此基础上做的封装扩展
    -  因为wheel事件，每个浏览器都不相同，需要做兼容处理   
         
浏览器 | 事件支持| 持详情
-------| --------  | ------- 
safari |     不支持wheel 事件| -  
     - | 支持mousewheel事件 |   wheelDeltaY , 往下是  -, 往上 + 
     - | 同时绑定wheel， mousewheel事件的触发情况|  wheel事件不触发，  mousewheel 触发
chrome | 支持wheel 事件| deltaY，  往下是  +, 往上 -；（数值为100）
    - | 支持mousewheel事件 |   wheelDeltaY 属性 ,     往下是  -, 往上 + ； （数值为120）
    - |  同时绑定wheel， mousewheel事件的触发情况|   wheel事件触发，  mousewheel 不触发  
opera | 支持wheel 事件| deltaY，  往下是  +, 往上 -；（数值为100）
    - | 支持mousewheel事件 |   wheelDeltaY 属性 ,     往下是  -, 往上 + ； （数值为120）
    - |  同时绑定wheel， mousewheel事件的触发情况|   wheel事件触发，  mousewheel 不触发
ie | 支持wheel 事件| deltaY，  往下是  +, 往上 -；（数值为144）
    - | 支持mousewheel事件 |   wheelDeltaY 属性 ,     往下是  -, 往上 + ； （数值为120）
    - |  同时绑定wheel， mousewheel事件的触发情况|   wheel，  mousewheel 都触发
firefox | 支持wheel 事件| deltaY，  往下是  +, 往上 -；（数值为3）
    - | 不支持mousewheel事件 |  - 
    - | 支持DOMMouseScroll事件 |  属性为detail 往下是  +, 往上 -；（3）（唯独火狐支持）
    - |  同时绑定wheel， mousewheel事件的触发情况|    wheel事件触发，  mousewheel 不触发
    
因此选择同时绑定 wheel 和  mousewheel 事件。

### 3.组件功能实现

#### 3.1隐藏滚动条
- 隐藏滚动条，对浏览器分析，做兼容性处理
    - webkit 内核浏览器， 可以通过伪类直接隐藏滚动条
        ```
        // 直接隐藏滚动条， 
        //  现在发现只有 webkit内核浏览器支持，
        // 如：chrome ，safari， opera 
        ::-webkit-scrollbar {
                    width: 0;
                    height: 0;
                }
        ```
    - 非webkit内核，需要通过css，人为隐藏滚动条，需要配合js，动态设置margin-right和height
        ``` 
          <style>
                 /*外围容器*/
                 .fd-contain {
                     position: relative;
                     width:400px;
                     height:400px;
                     overflow: hidden;
                     border: 1px solid #000;
                 }
                 /*滚动条区域*/
                 .fd-scroll-area {
                     /*关键属性 父height + 滚动条大小17，隐藏横向滚动条（下滚动条）*/
                     height: 417px;
                     /*关键属性 隐藏纵向滚动条 （右侧滚动条）*/
                     margin-right: -17px;
                     /*关键属性， 容器强制出现滚动条*/
                     overflow: scroll;
                 }
                 p {
                     line-height:200px;
                 }
             </style>
             
            <div class="fd-contain">
               <div class="fd-scroll-area">
                   <p>111111111111111111111111</p>
                   <p>11</p>
                   <p>11</p>
                   <p>11</p>
                   <p>11</p>
                   <p>11</p>
                   <p>11</p>
                   <p>11</p>
               </div>
            </div>
    
        ```
#### 3.2 事件监听，设置模拟滚动条位置
- 监听浏览器的scroll事件
    - 获取其 scrollTop , scrollLeft
    - 根据其 （scrollTop : scrollHeight）和 scrollLeft : scrollWidth 的比值， 分别设置模拟滚动条的位置
    - 触发 scroll 事件
#### 3.3 通过拖动div模拟的滚动，反向设置滚动条的位置
- 鼠标按下，开始拖动，记录滚动条的开始位置
- 鼠标移动，开始移动，记录结束位置，设置模拟div的位置，根据状态，反向推算scrollTop位置
    - 移动频率较高，可以进行节流处理
- 鼠标松开，结束移动。设置滚动条位置
- pc和移动端事件兼容处理 
    - pc 端
        - mouse事件
             - mousedown
             - mousemove
             - mouseup
    - 移动端
        - touch事件
            - touchstart
            - touchmove
            - touchend
##### 3.4  其他  
- 更新滚动条位置 update
    - 根据值，设置滚动的位置
        - 设置模拟滚动条的状态
- 刷新方法 refresh
   - 会调用初始化方法  
       - 初始化参数 initParams
       - 设置模拟滚动条状态
- 响应式，设置滚动条的位置
    - 判断浏览器是否支持 mutationObserver， 内部实现，监听数据变更和属性变更，以及子节点变更
        - 支持的【自动】调用刷新（refresh）方法
        - 不支持的需要【手工】调用刷新（refresh）方法      
-  滚动条额外功能
   - pc 
        - 滚轮事件， 滚动到顶部/左边时触发   scrolltoupper()
        - 下滚触底，滚动到底部/右边时触发  scrolltolower()
   - 移动端
        - 滑动事件，下拉触顶，   scrolltoupper()
        - 滑动事件，上拉触底， scrolltolower()
        - *暂时未实现*
